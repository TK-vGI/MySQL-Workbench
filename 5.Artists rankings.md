# Stage 5/5: Artists rankings
## Description
In this final stage of the project, you will delve into advanced SQL queries using subqueries, aggregate functions,  
and window functions to analyze and rank artists based on the number of artworks they've created.

The primary question for this stage is to find the artists along with the number of artworks they've created and  
rank each artist based on the total number of artworks they've produced.

You have seen aggregate functions and subqueries in the previous stages. So, let's talk a bit about window functions.  
A window function executes an operation similar to an aggregate function but operates on a set of query rows individually.  
While an aggregate function combines query rows into a single result row, a window function generates a result for  
each query row. You will need the RANK window function to assign a rank to each artist.
